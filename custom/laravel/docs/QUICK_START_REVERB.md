# Quick Start: Laravel Reverb WebSocket Setup

This guide gets you up and running with Laravel Reverb WebSocket functionality in ClearLine quickly.

## 🚀 Development Setup (5 minutes)

### 1. Configure Environment

```bash
# Copy and configure environment
cp .env.example .env

# Generate application key
php artisan key:generate

# Generate Reverb credentials
php artisan reverb:install
```

### 2. Update .env File

```bash
# Database (use your preferred database)
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=clearline_dev
DB_USERNAME=root
DB_PASSWORD=

# Broadcasting
BROADCAST_CONNECTION=reverb

# Reverb WebSocket (generated by reverb:install)
REVERB_APP_ID=123456
REVERB_APP_KEY=your-generated-key
REVERB_APP_SECRET=your-generated-secret
REVERB_HOST=127.0.0.1
REVERB_PORT=8080
REVERB_SCHEME=http
```

### 3. Setup Database

```bash
# Run migrations
php artisan migrate

# Optional: Seed with sample data
php artisan db:seed
```

### 4. Start Services

```bash
# Terminal 1: Start Laravel
php artisan serve --host=127.0.0.1 --port=8000

# Terminal 2: Start Reverb WebSocket
php artisan reverb:start --host=127.0.0.1 --port=8080 --debug

# Terminal 3: Start queue worker (optional)
php artisan queue:work
```

### 5. Configure Frontend

```bash
# Frontend .env (custom/ui/svelte-ui/.env)
VITE_API_BASE_URL=http://127.0.0.1:8000
VITE_WS_URL=ws://127.0.0.1:8080/app/your-generated-key
```

### 6. Build and Deploy Frontend

```bash
cd custom/ui/svelte-ui
pnpm install
pnpm run build

# Copy to Laravel public directory
cp -r build/* ../laravel/public/app/
```

### 7. Test Setup

Visit `http://127.0.0.1:8000/app` and check browser console for WebSocket connection.

## 🔧 Production Setup

### 1. Environment Configuration

```bash
# Production .env
APP_ENV=production
APP_DEBUG=false
REVERB_SCHEME=https
REVERB_HOST=0.0.0.0

# Frontend production .env
VITE_API_BASE_URL=https://your-domain.com
VITE_WS_URL=wss://your-domain.com/app/your-production-key
```

### 2. Deploy with Script

```bash
# Make deployment script executable
chmod +x deployment/deploy.sh

# Run deployment
./deployment/deploy.sh
```

### 3. Configure Nginx

```bash
# Copy Nginx configuration
sudo cp deployment/nginx/clearline.conf /etc/nginx/sites-available/clearline
sudo ln -s /etc/nginx/sites-available/clearline /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 4. Setup Supervisor

```bash
# Copy supervisor configurations
sudo cp deployment/supervisor/*.conf /etc/supervisor/conf.d/
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start clearline-reverb:*
```

## 🐛 Common Issues

### WebSocket Connection Failed

```bash
# Check if Reverb is running
ps aux | grep reverb

# Check port is listening
netstat -tlnp | grep :8080

# Restart Reverb
php artisan reverb:start --host=127.0.0.1 --port=8080 --debug
```

### Authentication Errors

```bash
# Verify broadcasting auth route
curl -X POST http://127.0.0.1:8000/api/broadcasting/auth \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json"

# Check CORS configuration
php artisan config:show cors
```

### Frontend Not Loading

```bash
# Rebuild frontend
cd custom/ui/svelte-ui
pnpm run build
cp -r build/* ../laravel/public/app/

# Clear Laravel caches
php artisan optimize:clear
```
